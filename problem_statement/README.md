# 设计与实现不一致检查与识别

输入为目标代码仓路径和设计/RFC 文档路径。作品使用 AI agent 阅读设计、探索代码、
判断实际执行语义，并输出代码实现相对于设计规范的不一致问题与证据链。

约束：

- 不修改目标代码或设计文档。
- 对项目名、语言、框架和文档类型保持通用。
- 最终只发布经独立反证和真实源文件核验的 confirmed issue。
- `/result` 至少包含 `issues.json`、`issues.jsonl`、`00-summary.md` 和单 issue 报告。
- 每个 issue 包含差异描述、设计证据、代码证据、文件与行号、设计章节、差异原因、
  误报排除和置信度。
- 评价关注有效 issue 召回、误报率与六小时内完成检视。

公开样例答案不随作品提交，也不进入主检视链路。
